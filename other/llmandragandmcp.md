# LLM / RAG / MCP

## LLM（Large Language Model）

**大規模言語モデル**。ChatGPT や Claude の「頭脳」部分。

大量のテキストで学習した AI で、質問に答えたりコードを書いたりできる。ただし **学習時点までの知識しか持っていない**（社内ドキュメントや最新情報は知らない）。

---

## RAG（Retrieval-Augmented Generation）

**検索して、それをもとに回答を生成する**仕組み。

LLM の「知識が古い・社内情報を知らない」問題を解決する。

```
ユーザーの質問
    ↓
① 関連ドキュメントを検索（Retrieval）
    ↓
② 検索結果 + 質問を LLM に渡す
    ↓
③ LLM が回答を生成（Generation）
```

**例**：「この会社の休暇制度は？」と聞いたとき、社内規則 PDF を検索して LLM に読ませて回答させる。

---

## MCP（Model Context Protocol）

**LLM がツールや外部サービスを使うための共通規格**。Anthropic が策定。

LLM 単体では「文章を生成する」しかできないが、MCP を使うと外部とやり取りできるようになる。

```
LLM ── MCP ── ファイル読み書き
           ── データベース検索
           ── GitHub 操作
           ── カレンダー取得
           ── ...
```

USB のような規格で、**「MCP に対応したツール」なら LLM がどれでも使える**というのがポイント。

---

## 3つの関係まとめ

```
┌─────────────────────────────────────┐
│  LLM（頭脳）                         │
│   ↑ RAG で外部ドキュメントを注入     │
│   ↑ MCP で外部ツールを操作           │
└─────────────────────────────────────┘
```

- **LLM**：考える人
- **RAG**：必要な資料を渡す仕組み
- **MCP**：道具を使わせる仕組み

Mastra はこの3つを TypeScript で組み合わせてエージェントを作るフレームワーク。
