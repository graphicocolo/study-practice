### ④ 推論できない場合を作る

```ts
// 値を代入せずに変数を宣言する（初期値なし）
// → 型がどうなるか確認する
// → その変数に string を代入しようとしたらどうなるか
```

TypeScript は代入後に型を絞り込む（ナローイング）機能を持っています。any でも代入した値から型を追跡してくれます。

ただし any は型チェックを無効化するので、実際のコードでは避けるべき型です。tsconfig.json の strict モードを有効にすると any が暗黙的に使われたときにエラーにできます（noImplicitAny）。

## 確認ポイント

- ③で `const` と `let` の推論結果がなぜ違うのか、自分なりに説明してみる

**自分の説明**

`let` は再代入可能なので、ここでは文字列が入ることだけは推論されたため `let varString1: string` と推論された

↓

再代入できるので、将来どんな文字列が入るかわからない → string（広い型）

`const` は再代入不可なので、`"テキスト1"` の値そのものが型として推論された

↓

絶対に変わらないと確定している → "テキスト1"（値そのものが型）<br>
この "テキスト1" のような型を リテラル型 と言う
