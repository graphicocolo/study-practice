## 21：FizzBuzz判定関数

2回目の実装（`count % 15` で FizzBuzz を先に判定）はすでに定番の正しいアプローチ。

### アプローチA：文字列を「連結」して組み立てる

`""` から始めて Fizz と Buzz を足していき、最後に空なら数字を返す。`if` の入れ子をなくせる。

```ts
// ヒント：こんな流れ
let result = "";
if (n % 3 === 0) result += "Fizz";
if (n % 5 === 0) result += "Buzz";
// result がまだ "" なら数字を使う
```

### アプローチB：配列と `map` で書く

`for` ループを使わず、配列を生成して `map` で変換する関数型スタイル。

```ts
// 1〜100 の配列を作る
Array.from({ length: 100 }, (_, i) => i + 1)
// → map で各数値を FizzBuzz に変換する
```

**`Array.from` 第二引数の `_` について**

マップ関数は引数を2つ受け取る。

| 引数 | 内容 |
|---|---|
| 第一引数 `_` | 配列の要素の値（`undefined` が入っているが使わない） |
| 第二引数 `i` | インデックス（0始まり） |

`{ length: 100 }` から作った配列の要素は `undefined` が100個並んでいるだけなので第一引数は使わない。JavaScript の関数は使わなくても引数を省略できないため、`_` という名前で「意図的に無視している引数」であることを示す慣習になっている。

### アプローチC：判定ロジックをデータで持つ

条件と結果のペアを配列で持っておき、ループで判定する。条件が増えても本体を変えなくていい。

```ts
// ヒント：こんなデータ構造
const rules = [
  [15, "FizzBuzz"],
  [3, "Fizz"],
  [5, "Buzz"],
];
```
