# React JSX

## `{}` の中には式しか書けない

JSX の `{}` は「値として埋め込む場所」なので、**式（expression）** しか書けない。

| | 定義 | 例 |
|---|---|---|
| 式（expression） | 評価すると値になる | `1 + 2`、`x > 0 ? "正" : "負"`、`isView(value)` |
| 文（statement） | 処理の手順を記述する（値を返さない） | `if`、`switch`、`for` |

```tsx
// ❌ switch は「文」なので書けない
{ switch (x) { case ...: } }

// ✅ 三項演算子は「式」なので書ける
{ x === 'A' ? <A /> : <B /> }

// ✅ オブジェクトマップも「式」なので書ける
{ MAP[x] }
```

### `=` との関係

```js
x = 5      // 代入式（5 という値を返す → 式）
let x = 5  // 変数宣言文（値を返さない → 文）
```

`=` の有無ではなく、**値を返すかどうか**が式と文の違い。
